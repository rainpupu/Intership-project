### 各类别召回率（最佳模型 epoch 70）

| 类别 | 召回率 | 精确率 | 实例数 | 正确检测 |
|------|:-----:|:-----:|:------:|:--------:|

| Frightened | **41%** | 41% | ~47 | ~19 |

### 混淆矩阵分析

从混淆矩阵可以观察到：

- **Sleepy（66%）** 和 **Beg（61%）** 是识别最好的两类，可能与它们有较明显的面部/姿态特征有关
- **Scare（13%）** 识别最差，仅 9/70 正确检出，大量被误分为其他情绪
- **Anger（36%）** 和 **Wonder（34%）** 也有大量混淆
- 多数错误集中在**外观相似的情绪对**之间（如 Scare ↔ Frightened, Anger ↔ Scare）

## 三、模型性能分析

### 检测速度

| 阶段 | 耗时 |
|------|------|
| 预处理 | 0.7ms / 张 |
| 推理 | 20.0ms / 张 (GPU) |
| 后处理 | 1.4ms / 张 |
| **总计** | **~22ms / 张** |

### 当前模型的主要局限

1. **整体准确率偏低（~46.5%）**：7 类情绪区分难度大，远低于健康检测模型的 92.3%
2. **Scare 类几乎无法识别**（13% 召回率）：大部分被误分为 Anger 或 Frightened
3. **最佳 Recall 在第 3 轮即达到峰值**，后续轮次没有提升，说明模型难以学到更具判别力的特征
4. **mAP@50:95 仅 33%**，框的定位和分类精度都不理想
5. **各类别间精度差异大**（13%~66%），模型对不同情绪的处理能力不均衡

### 原因分析

- **情绪表达差异细微**：猫的情绪主要通过耳朵角度、眼睛形状、胡须方向等细节表达，YOLO11n 的特征提取能力有限
- **类间相似度高**：Scare 和 Frightened、Anger 和 Scare 在视觉上难以区分
- **类别不平衡**：Sleepy（166 张）是 Scare（70 张）的 2.4 倍，模型偏向于多数类
- **数据质量**：情绪标注具有一定主观性，可能存在标注不一致

## 四、训练过程

### 训练配置

| 参数 | 值 |
|------|-----|
| 基础模型 | YOLO11n（猫品种检测模型迁移学习） |
| 输入尺寸 | 640×640 |
| 批次大小 | 8（6GB 显存友好） |
| 优化器 | AdamW（自动选择） |
| 初始学习率 | 0.01 → 0.001667（自动修正） |
| 学习率策略 | 余弦退火（cos_lr） |
| 预热轮数 | 3 |
| 早停耐心 | 20（第 90 轮触发） |
| 数据增强 | mosaic, flip, auto-augment, erasing |
| 混合精度训练 | 开启 (AMP) |

### 训练过程特点

- **早停触发**：训练在 90 轮停止（patience=20），后续指标无提升
- **Recall 峰值出现在第 3 轮**，之后震荡不再突破，说明模型很快陷入瓶颈
- **验证损失在第 30 轮后趋于平稳**，无明显过拟合，但也无明显提升
- **整体精度偏低**，分类损失（cls_loss）从初始 3.23 下降到 0.76，仍有较大下降空间

## 六、结论与改进建议

### 结论

1. **猫情绪分类任务难度较大**，当前模型整体 mAP@50 仅 **55.5%**，远低于品种检测和健康检测
2. **Sleepy（66%）和 Beg（61%）** 识别效果相对较好，**Scare（13%）** 基本无法识别
3. **迁移学习效果有限**：猫品种检测的预训练权重对情绪这种细粒度特征的帮助不大
4. **早停在 90 轮触发**，模型在 30 轮后已无实质提升

### 改进建议

- **改用更大的模型**：YOLO11s 或 YOLO11m 可能提取更细粒度的特征
- **增加训练数据**：尤其是 Scare（当前仅 ~70 张）和 Anger 等低召回类别
- **合并相似类别**：考虑将 Scare 与 Frightened 合并，Anger 与 Scare 区分度过低
- **使用分类专用模型**：情绪识别更适合用分类模型（如 ResNet/EfficientNet）而非目标检测模型
- **数据增强优化**：增加针对耳朵、眼睛等局部特征的裁剪增强
