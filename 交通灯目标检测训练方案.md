## 二、train.json 与 eval.json 的作用

### train.json - 训练集标注文件

- **格式**：标准 COCO JSON，包含 `images`、`annotations`、`categories` 三个顶层字段
- **内容**：1781 张行车记录仪拍摄的交通场景图像，每张图像含有 1 个或多个交通灯的边界框标注
- **用途**：提供 YOLOv11 模型训练所需的输入图像和真实标注（ground truth），模型通过反向传播学习交通灯状态分类和定位

### eval.json - 验证集标注文件

- **格式**：与 train.json 完全一致的 COCO JSON 结构
- **内容**：445 张独立于训练集的交通场景图像及其标注
- **用途**：在每个训练 epoch 结束后对模型进行验证，计算 mAP、precision、recall 等指标，用于监控过拟合和选择最佳模型权重（best.pt）

### COCO 标注结构示例

```json
{
  "images": [
    {"id": 13, "file_name": "00013.jpg", "width": 1920, "height": 1080}
  ],
  "annotations": [
    {
      "image_id": 13,
      "category_id": 7,
      "bbox": [1017, 740, 46, 46],
      "area": 2116,
      "iscrowd": 0,
      "segmentation": []
    }
  ],
  "categories": [
    {"id": 4, "name": "Traffic Light-Red Light"},
    {"id": 5, "name": "Traffic Light-Yellow Light"},
    {"id": 6, "name": "Traffic Light-Green Light"},
    {"id": 7, "name": "Traffic Light-Off"}
  ]
}
```

### 3.2 核心实体关系（训练→检测闭环）

```
DetectionScene (检测场景，如 "traffic_light")
  │
  ├── 1:N → TrainingTask
  │         ├── 训练完成后 → ModelVersion (best.pt, mAP等指标)
  │         └── 每epoch → TrainingMetric (loss, mAP, precision, recall)
  │
  ├── 1:N → ModelVersion
  │         └── is_default=True → 检测时使用的默认模型
  │
  └── 1:N → DetectionTask
            └── 1:N → DetectionResult (每个检测框)
```

### 3.3 关键数据流：COCO数据集 → 模型训练 → 检测部署

```
train.json ──→ convert_coco_to_yolo()  ──→ labels/train/*.txt
eval.json  ──→ convert_coco_to_yolo()  ──→ labels/val/*.txt
                                              │
                    generate_data_yaml() ←────┘
                         │
                         ▼
                    data.yaml
                         │
                         ▼
              TrainingService.start_training()
                         │
                    YOLO('yolov11n.pt').train(data='data.yaml', ...)
                         │
                         ▼
               runs/train/task_N/weights/best.pt
                         │
                         ▼
              TrainingService.validate_model()
                         │  → mAP50, mAP50-95, precision, recall, per_class_AP
                         │  → 写入 ModelVersion 表
                         ▼
              DetectionService.load_model(scene_id)
                         │
                         ▼
              DetectionService.detect_single/batch/video/camera
                         │
                         ▼
              DetectionResult (边界框 + 类别 + 置信度) → 前端 Canvas 渲染
```

## 四、交通灯目标检测训练框架

### 4.1 类别映射

COCO category_id 与类别标签之间的映射关系：

| COCO ID | YOLO class_id | 英文名 | 中文名 |
|---------|---------------|--------|--------|
| 4 | 0 | Traffic Light-Red Light | 红灯 |
| 5 | 1 | Traffic Light-Yellow Light | 黄灯 |
| 6 | 2 | Traffic Light-Green Light | 绿灯 |
| 7 | 3 | Traffic Light-Off | 未亮灯 |

### 4.2 推荐训练超参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `model_name` | `yolov11n` | 轻量模型，适合黄灯样本极少的场景，降低过拟合风险 |
| `epochs` | 100 | 标准训练轮数 |
| `img_size` | 640 | COCO 预训练权重默认尺寸 |
| `batch_size` | 16 | GPU 内存充足时可用 16，不足时降至 8 或 4 |
| `lr0` | 0.01 | SGD 初始学习率 |
| `optimizer` | SGD | YOLO 默认优化器 |
| `device` | `cuda:0` | 有 GPU 优先；CPU 训练耗时约 10-20 倍 |

### 4.3 实施步骤

#### Step 1：COCO 格式 → YOLO 格式转换

利用已有的 `convert_coco_to_yolo()` 函数（位于 `backend/app/services/data_utils.py` 第 210 行），将 COCO JSON 转换为 YOLO TXT 格式。

```python
# 调用方式
convert_coco_to_yolo(
    coco_json_path="e:/program/archive/trafficlight/train.json",
    output_dir="datasets/trafficlight/labels/train",
    image_dir="datasets/trafficlight/images/train"
)
convert_coco_to_yolo(
    coco_json_path="e:/program/archive/trafficlight/eval.json",
    output_dir="datasets/trafficlight/labels/val",
    image_dir="datasets/trafficlight/images/val"
)
```

> **注意**：当前 `convert_coco_to_yolo` 使用 `enumerate()` 生成连续 class_id（0,1,2,3），与 COCO 原始 ID（4,5,6,7）无关，类别顺序由 `categories` 数组顺序决定，正好一一对应。

#### Step 2：组织图像文件

将 `e:/program/archive/trafficlight/` 下的原始图像文件（.jpg）按 train/eval 分别复制到对应目录：
- `datasets/trafficlight/images/train/` — 训练集图像
- `datasets/trafficlight/images/val/` — 验证集图像

#### Step 3：生成 data.yaml

利用已有的 `generate_data_yaml()` 函数（位于 `backend/app/services/data_utils.py` 第 431 行）。

```yaml
# 生成内容示例
path: datasets/trafficlight
train: images/train
val: images/val
test: images/val

nc: 4
names:
  0: Red
  1: Yellow
  2: Green
  3: Off
```

#### Step 4：注册 DetectionScene

在 VisAgent 平台注册一个交通灯检测场景：

```python
DetectionScene(
    name="traffic_light",
    display_name="交通灯检测",
    description="车载影像交通灯状态识别（红/黄/绿/未亮灯）",
    category="traffic",
    class_names=["Red", "Yellow", "Green", "Off"],
    class_names_cn={"Red": "红灯", "Yellow": "黄灯", "Green": "绿灯", "Off": "未亮灯"}
)
```

#### Step 5：创建并启动训练任务

通过 API 或前端界面创建训练任务：
- 选择场景：交通灯检测
- 选择模型：yolov11n
- 设置超参数（epochs=100, batch_size=16, img_size=640, lr0=0.01, optimizer=SGD）
- 指定 data.yaml 路径和数据集路径
- 启动训练

#### Step 6：监控训练过程

- 通过 `GET /api/training/tasks/{task_id}/status` 轮询训练状态和进度
- 通过 `GET /api/training/tasks/{task_id}/metrics` 获取每个 epoch 的 loss 和 mAP 曲线
- 前端 ECharts 实时渲染 loss/mAP 趋势图

#### Step 7：模型验证

```python
# POST /api/training/tasks/{task_id}/validate
# 返回指标：
{
    "mAP50": 0.xxx,
    "mAP50-95": 0.xxx,
    "precision": 0.xxx,
    "recall": 0.xxx,
    "f1": 0.xxx,
    "per_class_ap": {
        "Red": 0.xxx, "Yellow": 0.xxx, "Green": 0.xxx, "Off": 0.xxx
    }
}
```

#### Step 8：部署检测

训练完成且模型版本注册后，自动作为该场景的默认模型（`is_default=True`），即可通过检测 API 进行推理：

- 单图检测：`POST /api/detection/single`
- 批量检测：`POST /api/detection/batch`
- 视频检测：`POST /api/detection/video`
- 摄像头实时检测：WebSocket `/api/camera/detect`

## 五、类别不平衡应对策略

黄灯（Yellow）仅占总样本 1.7%，需采取措施：

| 策略 | 实现方式 | 备注 |
|------|----------|------|
| 数据增强 | YOLOv11 默认启用 mosaic、hsv、flip 等 | 无需额外配置 |
| 类别权重 | 在 `model.train()` 中未直接支持，可考虑修改损失函数 | 高级方案 |
| 降低过拟合 | 使用 yolov11n（最小模型）、early stopping | 推荐首选 |
| 合成样本 | 可通过颜色变换将红灯/绿灯图像合成为黄灯 | 数据预处理阶段 |
